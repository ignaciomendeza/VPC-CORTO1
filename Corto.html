La Opción B se estandarizó porque varias convoluciones pequeñas (3×3) pueden lograr un campo receptivo equivalente a filtros más grandes, pero introduciendo más capas no lineales (ReLU) entre ellas. Este diseño se hizo bastante popular por arquitecturas como VGGNet, que fue propuesta por Karen Simonyan y Andrew Zisserman.

Task 2¶

Inciso 1

Los métodos tradicionales como SIFT y HOG dependen de descriptores de características diseñados manualmente que encuentran patrones locales de bordes o gradientes. Aunque pueden funcionar en entornos controlados, en un contexto agrícola real pueden ser limitantes, ya que las hojas pueden aparecer con condiciones distintas de iluminación, diferentes ángulos en los que se tomó la foto, o tapadas en parte por otras hojas o ramas. Estas variaciones cambian bastante los gradientes y puntos clave que estos algoritmos usan, lo que hace que haya una pérdida de consistencia en los descriptores y disminuye la capacidad de un clasificador como un SVM para identificar correctamente las enfermedades.

Por otro lado, los modelos de Deep Learning basados en redes convolucionales generan mapas de características jerárquicos que aprenden automáticamente representaciones visuales desde patrones simples hasta estructuras semánticas más complejas. En las primeras capas se encuentran bordes y texturas, mientras que en capas profundas se capturan patrones de alto nivel relacionados con la forma y las manchas que caracterizan una enfermedad. Esta representación jerárquica hace que el modelo sea mucho más completo con las variaciones de iluminación y perspectiva, ya que las características importantes se aprenden directamente de los datos en lugar de que sean definidas manualmente.

Inciso 2

Si tuviera que elegir entre AlexNet y VGG-16, elegiría VGG-16 porque su arquitectura es más profunda y modular, y está basada en múltiples convoluciones pequeñas de 3×3. Esta estructura permite aumentar poco a poco el campo receptivo y al mismo tiempo mantiene una organización consistente de bloques convolucionales, lo que hace más fácil el uso de transfer learning para reutilizar características aprendidas en grandes datasets. Además, la presencia de múltiples funciones de activación ReLU introduce más no-linealidad en la red, aumentando su capacidad para aprender representaciones visuales complejas y abstractas. Esto es útil para identificar patrones de enfermedades en hojas de cultivo.

def entrenar_una_epoca(modelo, loader, criterio, optimizador, device):
    modelo.train()
    perdida_total = 0.0
    correctas = 0
    total = 0

    barra = tqdm(loader, desc="Entrenando", leave=False)

    for x, y in barra:
        x, y = x.to(device), y.to(device)

        optimizador.zero_grad()
        logits = modelo(x)
        perdida = criterio(logits, y)
        perdida.backward()
        optimizador.step()

        perdida_total += perdida.item() * x.size(0)
        preds = logits.argmax(dim=1)
        correctas += (preds == y).sum().item()
        total += y.size(0)

        barra.set_postfix(loss=perdida.item())

    return perdida_total / total, correctas / total


@torch.no_grad()
def evaluar(modelo, loader, criterio, device):
    modelo.eval()
    perdida_total = 0.0
    correctas = 0
    total = 0

    barra = tqdm(loader, desc="Validando", leave=False)

    for x, y in barra:
        x, y = x.to(device), y.to(device)

        logits = modelo(x)
        perdida = criterio(logits, y)

        perdida_total += perdida.item() * x.size(0)
        preds = logits.argmax(dim=1)
        correctas += (preds == y).sum().item()
        total += y.size(0)

    return perdida_total / total, correctas / total

@torch.no_grad()
def obtener_preds_y_true(modelo, loader, device):
    modelo.eval()
    y_true = []
    y_pred = []

    for x, y in loader:
        x = x.to(device)
        logits = modelo(x)
        preds = logits.argmax(dim=1).cpu().numpy()

        y_true.extend(y.numpy())
        y_pred.extend(preds)

    return np.array(y_true), np.array(y_pred)

def plot_matriz_confusion(cm, clases, titulo="Matriz de confusión"):
    fig, ax = plt.subplots(figsize=(8, 6))
    im = ax.imshow(cm)

    ax.set_title(titulo)
    ax.set_xlabel("Predicción")
    ax.set_ylabel("Real")

    ax.set_xticks(np.arange(len(clases)))
    ax.set_yticks(np.arange(len(clases)))
    ax.set_xticklabels(clases, rotation=45, ha="right")
    ax.set_yticklabels(clases)

    for i in range(cm.shape[0]):
        for j in range(cm.shape[1]):
            ax.text(j, i, str(cm[i, j]), ha="center", va="center")

    fig.colorbar(im, ax=ax)
    plt.tight_layout()
    plt.show()

plot_matriz_confusion(cm, dataset.classes)
